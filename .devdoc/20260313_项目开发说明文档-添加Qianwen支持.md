# 20260313_项目开发说明文档-添加Qianwen支持

## 目的
为 AIchat2doc 插件添加阿里云通义千问 (`https://www.qianwen.com/` 和 `https://tongyi.aliyun.com/`) 原生界面的“保存为 Obsidian 笔记”与“下载 Markdown 文件”功能。

## 当前能够实现
- 能够通过解析千问原生的按钮区域（基于 `[data-icon-type="qwpcicon-copy"]` 定位）来安全注入额外的导出与下载按钮。
- 能够定位到对应的回答区域 `.tongyi-markdown` 并进行干净的 DOM 克隆和清理。
- 对通义千问特定的内部布局（如代码块内部掺杂行号类及额外的 UI 节点，如 `.react-syntax-highlighter-line-number` 行号 span）进行了专项 DOM 清除，确保输出的 Markdown 代码块整洁干净并具备正确的语言标记。

## 具体实现 
- 在 `manifest.json` 中加入了新的 Matches URL："https://www.qianwen.com/*" 和 "https://tongyi.aliyun.com/*" 下的权限匹配。
- 在 `content.js` 中的 `PLATFORMS` 新增 `qianwen` 字段：
  - `match()`: 检测域名含 `qianwen.com` 或 `tongyi.aliyun.com`。
  - `getCopyButtons()`: 查询所有原生复制操作的图标对应节点 `span[data-icon-type="qwpcicon-copy"]` 序列。
  - `getActionBar()`: 基于拷贝图标节点向外反向寻找并定位通用的工具按钮容器节点 `div[class*="leftArea"]`。
  - `inject()`: 生成匹配 Qianwen 原生界面的 tailwind `<button>` 节点并将其插入到操作区中。
  - `getContentNode()`: 通过向上溯源寻找同级范围中的内容容器 `.tongyi-markdown` 用于定位对话文本和代码。
  - `cleanDOM()`: 移除了代码块中的行编号节点（`.react-syntax-highlighter-line-number`），并将包含着真实代码的 `<code>.textContent` 完全提取赋给新建的只标记有格式信息 (`class="language-XXX"`) 的全新 `<code>` 中，这样完美规避了代码在 Turndown 处理过程中的新段落或换行错误，保证了 Markdown 的还原效果。

## AI 讨论记录
1. **特殊UI格式拦截分析**：经过分析得出通义千问在自带 UI 处理时会在每个 `<pre>` 外封装非常复杂的操作状态 UI。最明显的是原生采用了 `<span class="linenumber react-syntax-highlighter-line-number">` 以添加代码行号显示。若直接使用内建 `turndownService` 获取最终文本的话，则它会把行号和代码混淆。
2. **解决方案定调**：由于此时 `<code>.textContent` 可以完整拿到带空 span 的结构（而内含真实多行回车格式），在 `cleanDOM` 中提前主动将包含 `react-syntax-highlighter-line-number` 判断逻辑的 span 进行剔除。剩余的内容使用全新 `<code>` 重新组装覆盖后交由转换器，得以完美提纯代码块并保留多行结构信息。
