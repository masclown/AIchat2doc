# 项目开发说明文档 - 适配Kimi

## 项目当前状态

**当前能够实现：**
- 快速将 Gemini、DeepSeek 和 Kimi 的对话页面转换为 Markdown 格式。
- 点击按钮跳转到 Obsidian 并新建一条带时间戳的笔记。
- 点击按钮下载 md 文件到本地。
- 基于平台适配器模式提取并清理不同平台特定 DOM 结构。
- 完美适配 Kimi 的 `.segment-assistant-actions` 操作栏及 `.markdown` 内容节点，清理不必要的按钮。

**当前无法实现 / 未解决问题：**
- 需要更多的平台测试以确保完全兼容。

**项目结构：**
- `manifest.json`: 浏览器插件配制文件。增加了 Kimi 域名（`kimi.moonshot.cn` 和 `kimi.com`）。
- `content.js`: 核心逻辑。在 `PLATFORMS` 适配器中新增了 `kimi` 配置。

## AI 讨论记录
- **问题1**：用户希望在现有插件基础上追加对 Kimi 平台的支持。
- **解决方案1**：根据用户提供的 HTML 片段，发现操作栏具有 `class="segment-assistant-actions"` 标志，导出按钮可以根据 `name="Copy"` 的图标位置插入；外层包装了 `class="markdown"` 作为实际内容节点。通过扩充 `PLATFORMS` 变量，实现了针对 Kimi 平台选择器的高效匹配。
- **问题2（代码块头信息清理）**：Kimi 导出的代码块带有一个 `class="segment-code-header"`，其中有语言名称和“复制”按钮。如果不清理，这部分内容可能会干扰 Turndown 的转换。
- **解决方案2**：在 `cleanDOM` 方法中移除了整个 `.segment-code-header` 以及 `.simple-button`，仅保留下方纯净的包含类名标识的 `<code>` 标签段落供 Turndown 处理。
- **问题3（按钮显示空白及段后多余换行）**：追加的按钮虽然存在且可点击，但在 Kimi 页面上无法显示图标；另外导出的 MD 中段落间存在较多无用的空行。
- **解决方案3**：
  1. 经排查原版 Kimi 复制按钮的 SVG 的 `viewBox` 是 `0 0 1024 1024`，而注入的新路径只有 16x16 大小。通过强行在克隆出的按钮内重设 `svg.setAttribute('viewBox', '0 0 16 16');` 恢复图标渲染。
  2. 针对多次出现的冗余换行符，发现 Kimi 回复块内存在如 `<div data-v-xxx></div>` 这样的空白块级元素，致使其被 Turndown 解析时额外增加了不可见的双换行间距。在 `cleanDOM` 环节针对全体无子元素且无文本的 div 执行 `.remove()`，有效压缩整理了阅读排版密度。
- **问题4（列表内外空白行过多）**：用户反馈渲染出的 markdown 虽然换行少了些，但列表（`<ul>` / `<ol>`）项的段后换行依然存在，且列表与其前文段落之间显得不够紧凑。
- **解决方案4**：
  1. 通过排查得知，Kimi 内部的列表项通常是由 `<li><div class="paragraph">...</div></li>` 构成的。这种 `li` 内包含块级元素的写法会被 Turndown 强行使用双回车加四个缩进的空格来间隔每个列表项。因此在 `cleanDOM` 环节加入了同 Gemini 类似的逻辑，将 `li` 内部包含的 `div.paragraph` 或 `p` 全部无损解包转换成 `span` 标签。
  2. 在 `getMarkdownContent` 获取字符串最后阶段，通过正则表达式 `\n{2,}([ \t]*[-*+]\s+)` 和 `\n{2,}([ \t]*\d+\.\s+)` 的替换，强制将渲染出来的双回车列表前缀（例如 `\n\n* xxx`）压缩为单回车（`\n* xxx`），确保列表项与前一文本段落直接相连。
- **问题5（表格上方多余文字）**：在 Kimi 导出的 Markdown 中，表格上方始终附有“表格”两个字，降低了内容整洁度。
- **解决方案5**：该文字来源于原页面表格自带的悬浮操作栏容器 `<header class="table-actions">`。在 `cleanDOM` 中加入并执行 `clone.querySelectorAll('.table-actions').forEach(action => action.remove());`，干净利落地移除了由于界面组件而造成的文本溢出。
