# 项目开发说明文档

## 目的
为项目 AIchat2doc 增加对豆包 (Doubao.com) 网页端平台的支持，使用户能够在其上使用“导出为 Markdown”与“保存到 Obsidian”这两个功能扩展按钮。

## 当前能够实现
- 已支持在页面消息操作栏中动态添加自定义扩展按钮。
- 已能够准确定位并提取用户的文本消息及代码块，兼容提取行内样式。
- 代码块可以自动被 Turndown 提取相关的 language 语言环境声明。
- 能够净化页面冗余的 UI 元素（例如多余的外框、按钮行、表格抬头说明等），提取出干净无污染的 Markdown 文本文档。

## 当前无法实现 / 未解决问题
- 暂无针对目标站点的异常已知问题。

## 具体实现
1. **修改 `manifest.json`**：将 `"https://www.doubao.com/*"` 加入到了匹配数组 `matches`。
2. **在 `content.js` 的 `PLATFORMS` 配置字典中实现了豆包适配器对象**：
   - 使用 `window.location.hostname.includes('doubao.com')` 作为页面匹配判断依据 `match`。
   - 利用 `data-testid="message_action_copy"` 特征获取宿主元素的操作栏。
   - 使用标准的无序操作元素渲染方式构建了带有适配原网页风格（去除外部 ID 影响内联表现并重置 viewBox 大小适配 16*16 的内部 SVG 数据结构）的操作按钮元素 `inject` 方法逻辑。
   - 使用 `.table-header-qH9Ajf` 和 `.header-wrapper-Mbk8s6` 选择了需要清理的无文本价值的外置交互组件，移除这些块确保了生成纯净不含交互状态文案的 Markdown 节点内容。

## 项目结构
主要涉及两处现有文件的修改，未新增影响核心逻辑的新代码组件：
- `manifest.json`
- `content.js`

## AI 讨论记录
- **讨论内容：** 如何干净地提取豆包在进行代码生成或表格生成时掺杂在 DOM 中的扩展操作按钮文案（如代码组件带有的 bash 标签栏、运行、复制按钮等）。
- **问题：** 如果直接转换原本的 HTML Node 组件到 Turndown 进行内容格式化，将会导致导出的代码块开头多出 "bash复制运行" 之类的说明杂项文字。
- **解决方案：** 编写 `cleanDOM` 回调中关于 `.header-wrapper-Mbk8s6` 指定剔除的方法。由于包含其实际代码高亮特征名的是 `.language-*` 的外框包裹 pre 内置元素而非标题框，可以在不影响语言名称提取的前提下，安全地把整个操作 Header 给移除。同样对于表格标题也是采用选取父框移除的方式净化纯净度。
