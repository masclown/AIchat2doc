# AIchat2doc 项目开发说明文档 - 添加插件弹出设置界面

## 目的

为 AIchat2doc Chrome 扩展添加点击插件图标弹出的设置界面，让用户可以自定义 Obsidian 库名称并持久化保存。

## 当前能够实现

- ✅ 点击 Chrome 工具栏中的插件图标，弹出设置界面
- ✅ 设置 Obsidian 库名称（Vault Name）
- ✅ 设置自动保存到 `chrome.storage.local`，刷新后不丢失
- ✅ `content.js` 中的"Save to Obsidian"功能动态读取用户设置的库名称

## 当前无法实现

- ❌ Markdown 文件默认下载路径设置（计划后续实现）

## 具体实现

### 新增文件
| 文件 | 说明 |
|------|------|
| `popup.html` | 弹出设置界面 HTML 结构 |
| `popup.css` | 暗色主题样式（Obsidian 紫色风格） |
| `popup.js` | 设置加载/保存逻辑（使用 `chrome.storage.local`） |

### 修改文件
| 文件 | 变更内容 |
|------|----------|
| `manifest.json` | 添加 `storage` 权限、`action.default_popup` 配置，版本号升至 0.0.18 |
| `content.js` | 移除硬编码 `VAULT_NAME`，新增 `getVaultName()` 异步函数从 storage 读取；`saveToObsidian()` 和 `saveActionHandler()` 改为 async |

### 技术要点
- **存储机制**：`chrome.storage.local` 可在 content script 和 popup 中共享数据
- **异步读取**：`getVaultName()` 返回 Promise，在 `saveToObsidian()` 中 `await` 调用
- **默认值**：库名称默认为 `"Obsidian"`，与之前的硬编码行为一致

## AI 讨论记录

### 2026-03-23: 添加插件弹出设置界面

**问题**：Obsidian 库名称 `VAULT_NAME` 硬编码为 `"Obsidian"`，用户无法自定义。

**方案**：
1. 使用 Chrome Extension 的 `action.default_popup` 在工具栏图标点击时弹出设置页
2. 使用 `chrome.storage.local` 持久化保存设置
3. `content.js` 中通过 `getVaultName()` 异步读取设置值

**Chrome 限制说明**：
- Content script 无法直接控制浏览器下载目录路径
- 后续若需实现默认下载路径，需添加 `background.js` (service worker) + `downloads` 权限
