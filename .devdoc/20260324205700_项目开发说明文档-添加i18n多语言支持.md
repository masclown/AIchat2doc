# AIchat2doc 项目开发说明文档 - 添加 i18n 多语言支持

## 目的

为浏览器扩展添加国际化（i18n）多语言支持，使用 Chrome 扩展原生 `chrome.i18n` API，支持简体中文和英文，语言自动跟随浏览器 UI 语言设置。

## 当前能够实现

- ✅ 点击插件图标弹出设置界面
- ✅ 设置 Obsidian 库名称
- ✅ 文件名「组合模式」和「模板模式」自定义
- ✅ 设置界面实时预览生成的文件名
- ✅ 所有设置持久化保存（`chrome.storage.local`）
- ✅ **i18n 多语言支持：简体中文（默认）和英文**
- ✅ **语言自动跟随 Chrome 浏览器 UI 语言**
- ✅ **popup 设置界面所有文本可翻译**
- ✅ **content script 按钮标题、弹窗文本可翻译**

## 当前无法实现

- ❌ Markdown 文件默认下载路径设置（计划后续实现）
- ❌ 手动切换语言（当前跟随浏览器语言，无手动切换功能）

## 具体实现

### 文件变更

| 文件 | 操作 | 说明 |
|------|------|------|
| `_locales/zh_CN/messages.json` | **新增** | 简体中文语言资源文件（default_locale） |
| `_locales/en/messages.json` | **新增** | 英文语言资源文件 |
| `manifest.json` | 修改 | 添加 `default_locale: zh_CN`，标题/名称使用 `__MSG_*__` 语法，版本号 → 0.0.20 |
| `popup.html` | 修改 | 为所有文本元素添加 `data-i18n` / `data-i18n-placeholder` 属性 |
| `popup.js` | 修改 | 新增 `applyI18n()` 函数；硬编码文本改为 `chrome.i18n.getMessage()` 调用 |
| `content.js` | 修改 | 按钮标题、错误日志、alert 弹窗、剪贴板提示、文件名默认文本均改为 i18n 调用 |

### i18n 技术方案

| 层级 | 方式 | 说明 |
|------|------|------|
| manifest.json | `__MSG_keyName__` 语法 | Chrome 原生支持，无需 JS 介入 |
| popup.html | `data-i18n` + JS 注入 | `applyI18n()` 遍历属性调用 `chrome.i18n.getMessage()` |
| popup.js | `chrome.i18n.getMessage()` | 状态提示、预览占位符等动态文本 |
| content.js | `chrome.i18n.getMessage()` | 按钮 tooltip、alert、console.error 等 |

### 翻译 Key 清单

共 30+ 个翻译键值对，涵盖：
- 扩展名称和标题（`extName`, `extTitle`）
- 设置界面标签和选项（`vaultNameLabel`, `fileNameModeLabel`, `modeCombined` 等）
- 状态提示（`saveSuccess`, `saveFailed`, `vaultNameRequired`）
- Content Script 文本（`saveToObsidian`, `downloadAsMd`, `chatLog`, `clipboardNotice`）

## AI 讨论记录

### 2026-03-24: 添加 i18n 多语言支持

**需求**：为浏览器插件增加国际化多语言支持。

**方案**：
- 使用 Chrome 扩展原生 `chrome.i18n` API，语言自动跟随浏览器 UI 语言
- 创建 `_locales/zh_CN/messages.json`（默认）和 `_locales/en/messages.json`
- `manifest.json` 中使用 `__MSG_*__` 语法替换硬编码文本
- `popup.html` 中通过 `data-i18n` 属性标记待翻译元素，由 `popup.js` 中的 `applyI18n()` 函数在 DOMContentLoaded 时自动注入翻译
- `content.js` 中所有用户可见文本（按钮标题、alert、剪贴板提示、默认文件名等）均改为 `chrome.i18n.getMessage()` 调用
- `saveFailed` 消息使用 Chrome i18n 的 `$placeholder$` 机制传递动态错误信息
